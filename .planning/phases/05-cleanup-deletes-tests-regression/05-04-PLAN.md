  <action>
    Edit `seers_harness/workflow/dag_runner.py`. Add a public method
    `run_request` on `WorkflowRuntime`:

    ```python
    def run_request(
        self,
        *,
        scenario: Any,
        nodes: list[NodeSpec],
    ) -> dict[str, Path]:
        """Drive a multi-node DAG sequentially.

        Calls `_run_node` per `nodes` entry in list order. After each
        node succeeds, reads the written artifact JSON back from disk
        and accumulates into a `dependency_payloads` dict keyed by
        `node.id` so the next node's `provider_payload_for_node` can
        resolve upstream artifacts.

        Returns: dict mapping `node.id` -> output_path.
        """
    ```

    Implementation rules:
    - Iterate `nodes` in order. For each `node`, call `self._run_node(...)`.
      `_run_node` already builds the per-node payload via
      `provider_payload_for_node(node_id, scenario, dependency_payloads,
      session_id)` — but the current `_run_node` body passes
      `dependency_payloads={}` (line 72-73). Refactor `_run_node` to
      accept a `dependency_payloads` keyword that defaults to `{}`, so
      `run_request` can pass an accumulated dict. Internal callers of
      `_run_node` that don't pass it (none currently) keep working.
    - After `_run_node` returns the output path, load JSON via
      `json.loads(output_path.read_text(encoding="utf-8"))` and store the
      parsed dict at `dependency_payloads[node.id]`.
    - Wrap the loop in a try/except: on any node failure, raise the
      original exception unchanged (do NOT swallow). The smoke test
      below asserts on raised exceptions for the failure path.
  </action>
