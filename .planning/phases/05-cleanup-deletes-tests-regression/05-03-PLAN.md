# Plan 05-03: `extra="forbid"` Schema 收紧

## Objective

把 `seers_harness/domain/models.py` 的 9 处 `model_config = {"extra":
"ignore"}` 一次性翻为 `{"extra": "forbid"}`，让任何未声明字段的 LLM 输出
立即抛 `ValidationError`，而不是被 `extra="ignore"` 静默吞掉。这是
ADR-01-PRINCIPLE-08（无 c14/c15 兼容包袱）和 ADR-01-PRINCIPLE-10（无 LLM
自评分字段）的结构性执行。

依赖 Wave 1：plan 05-01 改写过 `models.py` 的 file-level docstring
（移除 "legacy c14/c15 JSON on disk decodes silently" 描述），让 docstring
与翻转后的语义一致；plan 05-02 删除残留的 hard_check 路径，避免任何
fixture 在翻转后因为意外字段抛错。

## must_haves

- truths:
  - `models.py` 的 9 处 `model_config` 全部为 `{"extra": "forbid"}`
  - 122-test baseline + 1 新增（plan 05-02）= 123 测试全绿
  - 任何 LLM/handler 写入未声明字段的代码路径，到达 model_validate 时立即
    抛 `pydantic.ValidationError`
- forbid:
  - 仅翻 forbid 子集（部分 model 留 ignore）—— 必须 9 处全翻
  - 引入 `BaseModel` 子类继承式翻转 —— D-15 决定 `model_config` 留在每个
    BaseModel 上，不抽公共基类（让翻转 grep-able）
  - 删除任何字段定义 / 改动字段类型 / 改动 `field_validator` /
    `model_validator` 体
