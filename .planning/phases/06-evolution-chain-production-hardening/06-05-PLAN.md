# Plan 06-05: Promotion Public-Entry Smoke And Phase Closeout

## Objective

Audit promotion behavior into a deterministic current-workspace dry run and
smoke public entry points against current fixtures. This plan proves import,
build, and dry-run artifact writing under the current schema without real
promotion, live skill writes, runtime edits, or registry mutation.

## must_haves

- truths:
  - `seers_harness/evolution/promotion_smoke.py` imports cleanly.
  - Promotion smoke reads current workspace skill files and optional portfolio
    artifacts, then writes a dry-run report to the caller-provided output path.
  - The report states live skill writes are disabled in Phase 6.
  - Smoke tests prove current live skill files are unchanged before and after
    the dry run.
  - Final full-suite regression passes after plans 06-01..06-04.
- forbid:
  - Editing `../harness-runtime/`.
  - Writing to `workflow-skills/current/`.
  - Writing any champion registry, live promotion record, or release artifact.
  - Using old runtime promotion modules as implementation source.

<threat_model>
T-06-11: promotion smoke accidentally becomes live promotion.
Mitigation: tests hash live skill files before and after dry run; report explicitly sets writes disabled.

T-06-12: public entry smoke silently depends on old runtime schema.
Mitigation: smoke imports only workspace modules and current `workflow-skills/current/` files.
</threat_model>
