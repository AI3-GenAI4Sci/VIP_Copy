# Plan 06-01: Current-Schema Evolution Contracts And Tool-Use Delta Distillation

## Objective

Create the current workspace evolution surface without copying old runtime
evolution flows. This plan introduces strict delta contracts, rewrites
`distill-skill-deltas` as a tool-use skill with handlers, adds a docs-only
reference v2 schema design, and audits field names against the current schema.

## must_haves

- truths:
  - D-03: `distill-skill-deltas` is rewritten as a true tool-use skill with matching handlers; the model proposes observations and changes, while handlers enforce structure, evidence references, privacy scans, and final submit.
  - `workflow-skills/evolution/distill-skill-deltas/SKILL.md` exists and uses the current tool-use skill style.
  - `compare-champion-bundles` and `select-seed-probes` are absent from the workspace live skill surface.
  - Delta proposal handlers accept evidence-backed observations and proposed changes, not model self-rated metrics.
  - Reference v2 is a design document only; no emitter or live v2 artifact writer is added.
  - Current-schema field names use delta/proposal/trial/trajectory/portfolio/belief language, not old candidate/champion labels.
  - D-25: module placement stays small and cohesive, with focused tests rather than a broad service graph.
- forbid:
  - Editing `../harness-runtime/`.
  - Copying old runtime evolution modules into workspace.
  - Adding `confidence`, `score`, `probability`, `uncertainty`, or `strength` as model-emitted tool args or Pydantic fields.
  - Live skill overwrite or promotion.
  - Reintroducing `generate_json`, polling, hard-check gates, or compatibility schema slots.

<threat_model>
T-06-01: old runtime compare/select flows return as live workspace behavior.
Mitigation: source audit asserts obsolete skill names and champion/probe vocabulary are absent from workspace live surfaces.

T-06-02: private trace or user-state text leaks into durable delta records.
Mitigation: handler privacy scan blocks private terms and requires evidence refs instead of raw private payload.

T-06-03: model self-rated metrics masquerade as posterior belief.
Mitigation: tool args reject self-rated metric fields; portfolio code computes belief from trial outcomes later.
</threat_model>
